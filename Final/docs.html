Guía de uso, gramática, ejemplos y fases del compilador

1. Uso rápido

  1. Escribe una línea de código en la caja de texto (p. ej. num = 4).
  2. Haz clic en Compilar o presiona Enter (Ctrl/Cmd+Enter acelera).
  3. Observa las salidas de análisis Léxico, Sintáctico, Semántico y la Validación.

2. Gramática soportada

stmt   -> IDENT "=" expr
expr   -> term (("+"|"-") term)*
term   -> factor (("*"|"/") factor)*
factor -> NUMBER | IDENT | "(" expr ")"

IDENT: comienza con letra o "_" y puede contener letras, dígitos o "_". NUMBER: enteros o decimales.

3. Reglas semánticas

  • El RHS de una asignación debe evaluar a tipo number.
  • Las variables usadas deben haberse asignado antes (o en esta misma ejecución si mantienes el estado).
  • Operadores + - * / solo entre números.

4. Ejemplos

  • num = 4 ✓ válido
  • x = 2 + 3*5 ✓ válido
  • y = (x + 1) * 2 ✗ error si x no está declarado previamente
  • 3 = 4 ✗ error sintáctico (izquierda debe ser IDENT)

5. Fases del análisis

5.1 Léxico

Convierte la cadena en tokens: IDENT, NUMBER, OP, EQUAL, LPAREN, RPAREN. Señala caracteres no reconocidos.

5.2 Sintáctico

Construye un AST según la gramática. Reporta tokens sobrantes, expresión incompleta, etc.

5.3 Semántico

Verifica tipos y uso de identificadores. Reporta variables no declaradas u operaciones inválidas.

6. Atajos y tips

  • Enter compila; Ctrl/Cmd+Enter compila desde cualquier parte del input.
  • Los tokens aparecen como chips animados; el árbol se actualiza con transición.
  • Si quieres conservar variables entre compilaciones, quita resetSymbols() de app.js.
